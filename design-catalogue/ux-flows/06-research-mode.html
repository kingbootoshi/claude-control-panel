Response

For Node.js rate limiting, there are a few solid approaches depending on your scale:

1. express-rate-limit - Simple, works for small to medium apps

  • In-memory by default, easy setup
  • Add redis store for distributed systems

2. Redis + sliding window - Better for high traffic

  • Use MULTI/EXEC for atomic operations
  • Consistent across multiple instances

3. API Gateway - Let infrastructure handle it

  • Kong, AWS API Gateway, Cloudflare
  • Offloads logic from your app

Code Example
import rateLimit from 'express-rate-limit'

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100, // limit each IP to 100 requests per window
  standardHeaders: true,
  legacyHeaders: false,
  message: { error: 'Too many requests' }
})

app.use('/api/', limiter)
// For production, add a Redis store for distributed rate limiting
