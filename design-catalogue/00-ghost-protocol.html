Here's the design philosophy:

GHOST-PROTOCOL.md
1. PROGRESSIVE DISCLOSURE - The terminal is primary. Everything else slides in. - No permanent sidebars for secondary features. - You access what you need, then it disappears. 2. PSYCHOLOGICAL CONFIDENCE - The interface doesn't explain itself. - No labels on the rail icons - they're obvious. - Status is shown, not announced. 3. SPATIAL HIERARCHY - Left rail: 64px, just icons - Main area: terminal + input - Everything else: slide-out panels 4. COLOR DISCIPLINE - Void black as base - Amber only for: active states, accents, prompts - Green only for: online/success - No other colors unless error

The result: an interface that feels like you're accessing something you shouldn't have access to. Every feature exists, but nothing competes for attention.
